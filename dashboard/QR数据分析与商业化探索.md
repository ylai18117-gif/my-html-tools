## 一、分析框架速查

### 1.1 数据源隔离原则

| 数据源 | 平台 | 提供的数据 | 用途 |
|--------|------|-----------|------|
| 广告平台 | toolsengine.careduka.com | 消耗、CPI、广告展示数、ECPM、广告首R | 获客成本、投放效率 |
| Firebase | Firebase Analytics | 新用户数、各页面到达人数、试用人数 | 用户漏斗、转化分析 |

**红线**：不用"消耗 ÷ Firebase新用户数"算CPI，也不用"广告平台安装数"算漏斗渗透率。两套数据各自独立使用。

### 1.2 三步分析法

**第一步 — 广告侧（先看钱）**

```
广告收入 = ECPM × 广告展示数 ÷ 1000
广告人均收入 = 广告收入 ÷ Firebase新用户数
人均展示次数 = 广告展示数 ÷ Firebase新用户数
```

健康度判断：
- ECPM 波动 > 30% → 检查广告源填充率和竞价变化
- 人均展示 < 3 次 → 广告场景太少或用户流失太快
- 广告首R > 15% → 广告侧健康
- 广告首R < 5% → 广告变现效率堪忧

**第二步 — 用户漏斗（再看过来的"人"）**

```
主页渗透率 = 主页展示 ÷ 新用户数 × 100%
试用率 = 试用人数 ÷ 新用户数 × 100%
步骤间流失率 = (上一步用户数 - 本步用户数) ÷ 上一步用户数 × 100%
```

健康度判断：
- 主页渗透率 < 30% → 新手流程广告/页面过重，需减负
- 试用率 < 5% → 订阅页曝光不足或页面转化力弱
- 语言选择页到假通知页流失 > 60% → 中间页面体验有问题

**第三步 — 二者结合（看回本）**

```
订阅缺口 = CPI（广告平台） - 广告人均收入
所需转订阅率 = 订阅缺口 ÷ (试用率 × 周价格 × 生命周期)
混合人均收入 = 广告人均收入 + 订阅人均收入
首日ROI = (混合人均收入 - CPI) ÷ CPI × 100%
```

回本状态判断：
- 首日ROI ≥ 0% → 已回本，可放量
- -20% ~ 0% → 接近回本，D2-D3广告可补回
- < -40% → 缺口过大，需优化漏斗或转化

### 1.3 续费率-生命周期对照表

| 一续率 | 生命周期（周） | 单用户LTV |
|--------|---------------|-----------|
| 60% | 6 | $89.94 |
| 50% | 5 | $74.95 |
| 40% | 4 | $59.96 |
| 30% | 2 | $29.98 |

### 1.4 关键杠杆排序（影响回本难度）

1. **试用率**（杠杆最大）— 翻倍则所需转订阅率减半
2. **一续率**（决定生命周期）— 30%→60%，LTV差3倍
3. **CPI**（获客成本）— 直接影响缺口大小
4. **ECPM/人均展示**（广告收入）— 缩小缺口但天花板有限
5. **转订阅率**（结果变量）— 受前4个影响

优化优先级：先稳试用率（>5%）→ 再保一续率（>50%）→ 然后压CPI → 最后提ECPM

## 三、阶段性分析结论

### 3.1 6.25-6.28 四天完整分析（2026-06-29）

**广告侧判断：**

ECPM 在 $26-$30 之间，波动幅度约14%，未触发30%异常线，广告填充和竞价环境稳定。广告首R 在 6.7%-9.0%，低于15%健康线但未触及5%危险线——"偏低但没崩"。

人均展示波动大（1.2~5.4次），6.27仅1.2次触发<3次警告。这与当天到达主页的用户量高度相关——到主页的人越少，可触发的广告场景就越少。人均展示的不稳定本质上是漏斗问题的下游表现。

**漏斗侧判断：**

两个卡口清晰可见。第一卡口是**语言选择页→假通知页**，转化率稳定在42%-56%，每2个人流失1个。第二卡口是**假通知页→主页**，6.28仅43.8%。两个卡口叠加导致主页渗透率四天全部低于30%警戒线（最高24.5%，最低13.5%），意味着超过3/4的新用户从未见过产品主页。

6.25是漏斗表现最好的一天：语言→假通知56.1%，假通知→主页50.7%，主页渗透率24.5%，试用率9.7%。但6.26-6.28三项指标全面回落，说明6.25的高转化可能受用户质量或素材因素影响，尚未稳定。

试用率四天累计仅15人（0+2+4+9），统计意义极弱。6.25的9.7%是唯一接近健康水位的数据点，但仅4人试用，不可作为基准。

**结合分析判断：**

6.25是唯一有希望的一天——试用率9.7%的情况下，如果一续率60%，仅需3.8%转订阅率即可回本，这完全可实现。但其他三天（试用率0%-4.3%）所需转订阅率飙升到8%-31%的不现实水平。

试用获取成本从$7.18（6.25）到$25.60（6.28），核心原因是试用率不稳定。这进一步证明当前优化重点不是变现端，而是让更多用户走到试用入口。

**核心结论：最大的杠杆在漏斗前端，不在变现端。**

当前不是"怎么让用户多付钱"的问题，而是"怎么让更多用户看到产品"的问题。主页渗透率每提升5个百分点，试用人数和广告展示数都会等比增长。

**6.25证明了模型可行性，但需要复现。** 关键问题：6.25的高转化是偶然还是可复制的？建议对照6.25和6.28的投放素材、时段、地区分布找差异。

**数据量不足以做精确决策。** 四天318个新用户、15个试用，转订阅数据为零。至少再积累2-3周数据，才能在试用率和转订阅率上得到统计上有意义的结论。当前重点：优化漏斗 + 稳定日新增用户量（目标>100/天）。

### 3.2 当前数据置信度评估

| 指标 | 置信度 | 原因 |
|------|--------|------|
| ECPM | 较高 | 样本量足够（广告展示数27-543） |
| CPI | 较高 | 广告平台直接数据 |
| 主页渗透率 | 中等 | 样本量52-143，有参考意义但波动大 |
| 试用率 | 低 | 单日试用人数0-9人，随机波动占主导 |
| 转订阅率 | 极低 | 尚无足够的付费转化数据 |

**结论**：当前阶段应重点关注漏斗指标（渗透率、试用率），积累更多数据后再做转订阅率的精确分析。

## 五、异常诊断清单

| 异常信号 | 可能原因 | 排查动作 |
|---------|---------|---------|
| 试用率突然下降 | 订阅页曝光被遮挡、新手流程变长 | 检查新手流程页面配置是否有变更 |
| 主页渗透率下降 | 广告加载过重、中间页面流失 | 对比变更前后各步骤渗透率 |
| ECPM大幅波动 | 广告源填充率变化、竞价环境变化 | 检查广告平台填充率和竞价报告 |
| CPI上升 | 投放素材疲劳、竞争加剧 | 检查广告素材CTR和竞价排名 |
| 一续率下降 | 产品周活价值不足 | 对比付费用户和免费用户的功能使用差异 |
| 广告首R很低 | 人均展示太少或ECPM下降 | 检查广告场景数和用户到达率 |
