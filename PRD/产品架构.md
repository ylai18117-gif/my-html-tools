# PhotoHtml 产品架构

## 产品定位

**一句话定位**：上传一张参考照片，AI 帮你拆解出「这张照片是怎么拍出来的」—— 从被拍人的姿势、摄影师的站位，到光线时机和构图技巧，给出一份可直接执行的拍摄指南。

**目标用户**：想拍出好看照片但缺乏专业知识的普通用户，尤其是经常在社交媒体上看到好看的参考图却不知道如何复刻的人群。

**核心价值**：把专业摄影师脑子里的「隐性知识」翻译成普通用户能看懂、能执行的具体指令。

---

## 问题场景

用户在网上看到一张很好看的照片（小红书/Instagram/朋友圈），想拍出类似效果但总是差很多。即使用同款手机、去同一个地点，结果依然不理想。

原因不是设备或场景，而是缺少以下知识：

- 被拍人该怎么站、怎么面向镜头
- 拍照的人应该站在什么位置、手机怎么拿
- 什么时间/光线下拍效果最好
- 用了什么构图技巧（三分法、引导线、框架等）
- 手机需要怎么设置（HDR、人像模式、曝光补偿等）

## 核心功能模块（MVP）

### M1：照片上传

- 支持拖拽上传、点击选择文件
- 支持粘贴图片（Ctrl+V）
- 支持粘贴图片 URL
- 上传后显示预览，确认后开始分析

### M2：AI 拍摄拆解分析

AI 对上传的照片进行多维度分析，输出结构化的拍摄指南：

**被拍人指南**
- 身体朝向与镜头的角度关系
- 姿势描述（站/坐/走/蹲，肢体语言）
- 面部朝向与表情建议
- 手部摆放建议

**摄影师指南**
- 拍摄距离（近景/中景/远景）
- 拍摄角度（平视/俯拍/仰拍，大约多少度）
- 手机持握方式（横屏/竖屏，单手/双手）
- 对焦位置建议

**光线与时机**
- 光线类型判断（自然光/人造光/混合光）
- 光线方向（顺光/逆光/侧光/顶光）
- 推荐拍摄时段（如黄金时段、阴天柔和光等）
- 影子利用建议

**构图技巧**
- 使用的构图法则（三分法/对称/引导线/框架/对角线等）
- 人物在画面中的位置比例
- 前景/背景元素利用
- 推荐焦距/镜头（手机上的等效设置）

**拍摄条件**
- 场景类型描述
- 天气/环境建议
- 人流量建议

**手机设置建议**
- 推荐拍摄模式（普通/人像/夜景/专业）
- HDR 建议
- 曝光补偿方向
- 其他设置建议

### M3：结果展示页

- 左侧展示原始参考图
- 右侧/下方分区展示各模块分析结果
- 每个分析点配以简洁的图标和示意图
- 整体布局清晰、信息分层，不堆砌

### M4：拍摄清单（Action Card）

- 将分析结果浓缩为 5-8 条可执行的拍摄步骤
- 卡片式设计，适合手机截图保存
- 用户可以带着这张清单去现场拍摄

## 用户流程

```
[首页]
  │
  ├─ 产品介绍（极简，一句话 + 示例）
  ├─ 上传入口（拖拽/点击/粘贴）
  └─ 示例展示（2-3 组 Before/After 分析案例）
  │
  ▼
[分析中]
  │
  ├─ 加载动画（不超过 10 秒）
  └─ 进度提示（"正在分析构图..." "正在拆解光线..."）
  │
  ▼
[分析结果]
  │
  ├─ 参考图展示
  ├─ 分区分析卡片（被拍人 / 摄影师 / 光线 / 构图 / 条件 / 设置）
  ├─ 拍摄清单（可截图保存的精简卡片）
  └─ 操作按钮（重新上传 / 分享结果）
```

## 技术选型（初步）

| 层面 | 方案 | 理由 |
|------|------|------|
| 前端 | 纯 HTML/CSS/JS 单文件 | 轻量、无需构建工具、快速验证 |
| AI 分析 | 视觉 AI API（如 Gemini Vision / Agnes） | 需要图像理解能力，输出结构化分析 |
| 样式 | 简洁摄影风，大图留白，杂志排版感 | 符合产品调性，提升专业感 |
| 部署 | 静态页面 | MVP 阶段无需后端 |

---

## 视觉风格

- **调性**：简洁、专业、摄影杂志感
- **配色**：黑白灰为主 + 一个点缀色（如暖橙或冰蓝）
- **排版**：大图留白，信息模块卡片化，呼吸感强
- **字体**：无衬线体为主，标题可适当用衬线体增加质感
- **动效**：克制，仅在关键交互点使用（如上传、分析完成）

---

## MVP 范围界定

**V1.0 包含**：
- 照片上传（拖拽 + 点击）
- AI 拍摄拆解分析（6 个模块）
- 分析结果展示页
- 拍摄清单卡片
- 响应式设计（移动端适配）

**V1.0 不包含（后续迭代）**：
- 用户账号/历史记录
- 对比模式（我的照片 vs 参考照片）
- 社交分享
- 多语言
- 拍照姿势骨架图叠加
- 基于地理位置的拍摄推荐
